# الفصل الثامن: المراجع (References)

## 🔄 الدرس الثالث والثلاثون: مراجعة هامة لتمرير القيم والمراجع (Call ByRef / ByVal)

هذا الدرس يُعتبر تمهيداً ومراجعة حاسمة لفهم كيفية تعامل لغة C++ مع الذاكرة (Memory)، وهو حجر الأساس لفهم المؤشرات (Pointers) لاحقاً.

عندما نقوم بتعريف متغير مثل int a = 10;، فإن نظام التشغيل يحجز مساحة (Slot) في الذاكرة (RAM) ويمتلك هذا المتغير 3 أشياء رئيسية:

الاسم (Name): وهو a.

القيمة (Value): وهي 10.

العنوان (Address): وهو عنوان سداسي عشر (Hexadecimal) يشير لمكان هذا المتغير في الذاكرة. (مثال: 0x7ffeeb42).

### 1. التمرير بالقيمة (Call By Value):

عند تمرير متغير إلى دالة بالطريقة العادية (بدون علامة &)، يقوم الكومبايلر بإنشاء متغير جديد تماماً في الذاكرة، ويقوم بنسخ القيمة من المتغير الأصلي إليه.

ماذا يحدث في الذاكرة؟ الدالة تتعامل مع "نسخة" (Copy) لها عنوان مختلف (Address مختلف).

النتيجة: أي تعديل يحدث داخل الدالة لا يؤثر أبداً على المتغير الأصلي في الـ main.

<div dir="ltr">

```cpp
#include <iostream>
using namespace std;

// الدالة تستقبل نسخة من المتغير (Call By Value)
void Function1(int x)
{
    x++; // سيتم زيادة النسخة فقط لتصبح 11
}

int main()
{
    int a = 10;
    Function1(a); // إرسال المتغير

    // القيمة الأصلية لن تتأثر
    cout << "Value of 'a' after Function1: " << a << endl; // النتيجة: 10
    return 0;
}
```

</div>

### 2. التمرير بالمرجع (Call By Reference):

عند إضافة علامة & قبل اسم المعامل في الدالة (مثل int &x)، فأنت تخبر الكومبايلر ألا يقوم بإنشاء أي نسخ جديدة في الذاكرة!

ماذا يحدث في الذاكرة؟ الكومبايلر لا يحجز مساحة جديدة، بل يجعل المتغير x بمثابة "اسم دلع" (Alias) للمتغير الأصلي a. كلاهما أصبح يشير إلى نفس العنوان (Address) ونفس المساحة في الذاكرة.

النتيجة: أي تعديل يحدث على x داخل الدالة، هو في الحقيقة تعديل مباشر على a الأصلية.

<div dir="ltr">

```cpp
#include <iostream>
using namespace std;

// الدالة تستقبل مرجع للمتغير الأصلي (Call By Reference)
void Function2(int &x)
{
    x++; // الزيادة ستحدث على المتغير الأصلي مباشرة
}

int main()
{
    int a = 10;
    Function2(a); // إرسال المتغير

    // القيمة الأصلية تغيرت!
    cout << "Value of 'a' after Function2: " << a << endl; // النتيجة: 11
    return 0;
}

```

</div>

🔍 كيف تطبع عنوان المتغير في الذاكرة؟

لتتأكد بنفسك من أن المتغير يمتلك قيمة وعنواناً، يمكنك استخدام علامة & (Address-of Operator) قبل اسم المتغير في جملة الطباعة لترى مكانه الحقيقي (الفيزيائي) داخل الرام.

<div dir="ltr">

```cpp
#include <iostream>
using namespace std;

int main()
{
    int a = 10;

    // طباعة القيمة (Value)
    cout << "Value of a: " << a << endl; // سيطبع: 10

    // طباعة العنوان في الذاكرة (Memory Address)
    cout << "Address of a: " << &a << endl; // سيطبع عنوان مثل: 00A7F85C

    return 0;
}
```

</div>

💡 الفائدة الهندسية:
التمرير بالمرجع (Call By Reference) لا يقتصر فقط على "تعديل" القيم، بل يُستخدم بشكل أساسي لتسريع البرنامج وتوفير الذاكرة، خاصة عند تمرير هياكل بيانات ضخمة (مثل Vectors و Structs) لتجنب عملية النسخ البطيئة والمستهلكة للذاكرة.

## 🔗 الدرس الرابع والثلاثون: إنشاء المراجع (Creating References)

في الدرس السابق فهمنا كيف تعمل المراجع عند إرسالها للدوال. الآن سنتعلم كيف ننشئ مرجعاً (Reference) لمتغير موجود داخل نفس النطاق (Scope) كطريقة لإعطاء المتغير اسماً آخر.

ببساطة، المرجع هو "اسم دلع" (Alias) لمتغير آخر. بمجرد ربط المرجع بمتغير، فإنهما يتشاركان نفس المساحة ونفس العنوان (Address) في الذاكرة.

كيف ننشئ المرجع؟

نستخدم علامة & عند تعريف المتغير الجديد: int &x = a;

ماذا يعني ذلك؟
الكومبايلر لا يقوم بإنشاء مكان جديد في الذاكرة للمتغير x، بل يجعله يشير مباشرة إلى نفس "الغرفة" التي يسكن فيها a.

النتيجة الحتمية:
بما أنهما اسمان لنفس المكان، فإن أي تغيير في قيمة x سيؤدي فوراً لتغيير قيمة a، والعكس صحيح!

💻 تطبيق عملي لفحص الذاكرة:

<div dir="ltr">

```cpp
#include <iostream>
using namespace std;

int main()
{
    int a = 10;

    // إنشاء مرجع x يؤشر على a
    int &x = a;

    // 1. طباعة العناوين (Addresses) للتأكد أنهما نفس المكان
    cout << "Address of a: " << &a << endl;
    cout << "Address of x: " << &x << endl;
    // النتيجة: سيطبع الكومبايلر نفس العنوان السداسي عشر تماماً للآثنين!

    // 2. طباعة القيم
    cout << "Value of a: " << a << endl; // 10
    cout << "Value of x: " << x << endl; // 10

    // 3. ماذا لو غيرنا قيمة المرجع x؟
    x = 20;

    cout << "\nAfter changing x to 20:" << endl;
    cout << "Value of a: " << a << endl; // ستصبح 20!
    cout << "Value of x: " << x << endl; // ستصبح 20!

    return 0;
}

```

</div>

💡 ملاحظة هامة جداً:
المراجع (References) في لغة C++ تختلف عن المؤشرات (Pointers) في أنه لا يمكن تغيير المتغير الذي يشير إليه المرجع لاحقاً. بمجرد أن تربط x بـ a، سيظل x اسماً آخر لـ a طوال فترة حياته، ولا يمكن جعله يؤشر على متغير آخر b مثلاً (وهذا ما سندرسه بعمق في فصل المؤشرات).
