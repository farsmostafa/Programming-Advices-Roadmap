# الفصل الثاني عشر: معالجة الاستثناءات (Exception Handling)

## 🛡️ الدرس التاسع والأربعون: التراي والكاتش (Try Catch)

في عالم البرمجة، الأخطاء (Errors) واردة جداً. بعض الأخطاء تكون (Logical Errors) أي أن البرنامج يعمل ولكنه يعطي نتائج حسابية خاطئة، وهناك أخطاء كارثية تحدث أثناء التشغيل (Runtime Errors) وتؤدي إلى انهيار البرنامج بالكامل وإغلاقه فجأة (Crash).

هنا يأتي دور الـ Exception Handling (معالجة الاستثناءات)، وهي طريقة لاحتواء هذا الانهيار ومنع البرنامج من الإغلاق في وجه المستخدم.

### 1. ما هو الـ Exception؟

الـ Exception هو حدث استثنائي أو خطأ غير متوقع يحدث أثناء عمل البرنامج (Runtime)، مثل:

محاولة الوصول لعنصر غير موجود في الـ Vector أو الـ Array.

محاولة قراءة File محذوف أو غير موجود على الهارد ديسك.

محاولة القسمة على صفر.

عندما يحدث هذا الخطأ، يقوم الكومبايلر بـ "رمي استثناء" (Throw Exception)، وإذا لم تقم بالتقاطه ومعالجته، سينهار البرنامج (Crash).

### 2. كيفية استخدام try و catch:

لاحتواء الانهيار، نضع الكود "الخطر" داخل بلوك يُسمى try، ونضع طريقة التعامل مع الخطأ داخل بلوك يُسمى catch.

try: "حاول تنفيذ هذا الكود، وإذا حدثت أي مشكلة (انفجار)، لا تُغلق البرنامج، بل ارسل المشكلة للـ catch."

catch: "أنا الحارس. إذا حدث الـ Exception في الـ try، سأقوم باصطياده وطباعة رسالة توضيحية بدلاً من عمل Crash للبرنامج."

💻 تطبيق عملي بالكود:

لنفترض أن لدينا Vector يحتوي على 5 عناصر، وحاولنا جلب العنصر السادس باستخدام دالة .at(5) التي ترمي Exception إذا كان الفهرس خارج الحدود (Out of Range).

<div dir="ltr">

```cpp
#include <iostream>
#include <vector>
using namespace std;

int main()
{
    vector<int> num = {1, 2, 3, 4, 5};

    // 1. الكود المحمي داخل try
    try
    {
        // هذه الجملة ستتسبب في كارثة (Crash) لأن الفهرس 5 غير موجود!
        cout << num.at(5) << endl;
    }
    // 2. صائد الأخطاء catch
    catch (...) // الثلاث نقاط (...) تعني اصطياد أي نوع من الـ Exceptions
    {
        // إذا حدث خطأ في الـ try، سيتم تجاهل الـ Crash وتنفيذ هذا الكود فقط
        cout << "Error: Out of bounds! (Exception Caught)" << endl;
    }

    // البرنامج سيكمل عمله بشكل طبيعي ولن يغلق فجأة
    cout << "Program is still running safely..." << endl;

    return 0;
}
```

</div>

🚨 تحذير هندسي هام جداً (Performance Warning):

رغم أن الـ try-catch تبدو كعصا سحرية لحل كل المشاكل، إلا أن المهندس أبو هدهود يُحذر بشدة من استخدامها بشكل مفرط.

السبب: عملية الـ Exception Handling تُبطئ البرنامج بشكل ملحوظ وتستهلك موارد الـ CPU والذاكرة.

الحل والـ Best Practice: لا تستخدم try-catch إلا في الضرورة القصوى (مثل التعامل مع الـ Files وقواعد البيانات). أما الأخطاء التي يمكن تفاديها بالمنطق العادي (مثل التأكد من حدود الـ Vector أو الـ Array)، فيجب عليك استخدام الـ Validation العادي (مثل جملة if) لتفادي المشكلة قبل وقوعها، هكذا:

<div dir="ltr">

```cpp
// الطريقة الأسرع والأفضل هندسياً (Validation بدلاً من Exception Handling)
int index = 5;
if (index >= 0 && index < num.size())
{
    cout << num.at(index) << endl;
}
else
{
    cout << "Error: Index out of range!" << endl;
}
```

</div>

استخدام الـ if لا يستهلك أي شيء من الـ Performance مقارنة بالـ try-catch. كن دائماً المبرمج الذي يعالج المشكلة قبل أن تحدث، وليس الذي ينتظر الانفجار ليحتويه!
