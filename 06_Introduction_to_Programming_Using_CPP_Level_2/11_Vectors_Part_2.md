# الفصل الحادي عشر: المتجهات - الجزء الثاني (Vectors Part II)

## 🔍 الدرس السادس والأربعون: الوصول إلى عناصر المتجه (Access Elements)

في هذا الفصل الجديد، سنقوم بتعميق فهمنا للمتجهات (Vectors). سنتعلم في هذا الدرس الطرق المختلفة للوصول (Access) إلى العناصر المخزنة داخل الفيكتور، والفرق الجوهري بينها.

بما أن المتجه (Vector) يُعامل معاملة المصفوفة، فإن الفهرس (Index) الخاص به يبدأ دائماً من الصفر (0).

### 1. طرق الوصول للعناصر:

لدينا طريقتان رئيسيتان لقراءة أو تعديل عنصر داخل الفيكتور:

طريقة الدالة المدمجة .at(): نستخدم دالة at ونمرر لها رقم الفهرس. (مثال: num.at(0)).

طريقة الأقواس المربعة [ ] (Array Syntax): نفس الطريقة الكلاسيكية المستخدمة مع المصفوفات العادية. (مثال: num[0]).

💻 تطبيق عملي بالكود:

<div dir="ltr">

```cpp
#include <iostream>
#include <vector>
using namespace std;

int main()
{
    // تعريف متجه وإعطاؤه 5 قيم ابتدائية (الفهرس من 0 إلى 4)
    vector<int> num = {1, 2, 3, 4, 5};

    cout << "--- Accessing using .at() ---\n";
    cout << "Element at index 0: " << num.at(0) << endl; // النتيجة: 1
    cout << "Element at index 2: " << num.at(2) << endl; // النتيجة: 3
    cout << "Element at index 4: " << num.at(4) << endl; // النتيجة: 5

    cout << "\n--- Accessing using [ ] ---\n";
    cout << "Element at index 0: " << num[0] << endl;    // النتيجة: 1
    cout << "Element at index 2: " << num[2] << endl;    // النتيجة: 3
    cout << "Element at index 4: " << num[4] << endl;    // النتيجة: 5

    return 0;
}
```

</div>

🚨 الفرق الجوهري والخطر الخفي (Out of Range):

ماذا يحدث لو كان الفيكتور يحتوي على 5 عناصر (من الفهرس 0 إلى 4)، وحاولت الوصول للفهرس رقم 5 (الذي لا وجود له)؟

عند استخدام دالة num.at(5): البرنامج سيقوم برمي خطأ استثنائي (Exception) ويتوقف عن العمل (Runtime Error). هذه الطريقة آمنة برمجياً لأنها تنبهك فوراً بوجود خطأ في حدود الفيكتور (Out of Range) لتتمكن من معالجته.

عند استخدام الأقواس num[5]: هنا تكمن الخطورة! الكومبايلر قد لا يعطيك خطأ صريح (Exception) يوقف البرنامج، بل قد يُكمل عمله ويعطيك مجرد تحذير (Warning) في وضع الـ Debug. أما في وضع التشغيل العادي، سيقوم بجلب بيانات عشوائية (Garbage Data) من الذاكرة (لأنك وصلت لمكان غير محجوز لك)، مما قد يسبب أخطاء حسابية وكوارث صامتة (Undefined Behavior) في نتائج برنامجك!

💡 نصيحة هندسية من المهندس أبو هدهود: دائماً احرص على عدم الخروج عن حدود الفيكتور أثناء البرمجة. وإذا كنت تكتب كوداً حساساً أو لست متأكداً من الحجم، يُفضل استخدام .at() لاكتشاف الأخطاء مبكراً.

## ✏️ الدرس السابع والأربعون: تعديل عناصر المتجه (Change Elements)

لقد تعلمنا كيف نقرأ العناصر، والآن حان الوقت لنتعلم كيف نُعدل (Update/Change) هذه العناصر، سواء بشكل جماعي باستخدام حلقات التكرار، أو بشكل فردي.

### 1. التعديل الجماعي باستخدام حلقة التكرار (Ranged Loop):

كما تعلمنا سابقاً، أفضل وأسرع طريقة للمرور على المتجه هي استخدام Ranged Loop مع المرجع (Reference &) لكي لا يقوم الكومبايلر بنسخ البيانات واستهلاك الذاكرة. ولكن هنا يظهر فرق جوهري:

للقراءة فقط (سريع وآمن): إذا كنت تريد طباعة المتجه فقط دون تعديله، يُنصح بشدة بإضافة كلمة const قبل المرجع. هذه الكلمة تضع "قُفلاً" (Lock) على العنصر وتمنعك برمجياً من تعديله بالخطأ.
for (const int &i : num)

للتعديل (سريع وقوي): إذا أردت تعديل جميع عناصر المتجه (مثلاً تصفيرها أو ضربها في 2)، نقوم بإزالة كلمة const، ونستخدم المرجع مباشرة int &i. في هذه الحالة، أي تغيير تجريه على المتغير i سينعكس فوراً على المتجه الأصلي!

### 2. التعديل الفردي للعناصر ([] و .at()):

بنفس الطريقة التي قرأنا بها العناصر الفردية، يمكننا استخدام الأقواس المربعة [] أو دالة .at() للوصول لعنصر معين وتغيير قيمته باستخدام علامة المساواة =.

(تذكير: دالة .at() أكثر أماناً لأنها سترمي Exception إذا حاولت تعديل عنصر خارج حدود المتجه).

💻 تطبيق عملي بالكود:

<div dir="ltr">

```cpp

#include <iostream>
#include <vector>
using namespace std;

int main()
{
    vector<int> num = {1, 2, 3, 4, 5};

    // 1. قراءة العناصر فقط (سريعة وآمنة بفضل const)
    cout << "--- Initial Vector ---\n";
    for (const int &i : num)
    {
        // i = 70; // ❌ خطأ: الكومبايلر سيرفض هذا التعديل لأننا استخدمنا const
        cout << i << " ";
    }
    cout << "\n\n";

    // 2. تعديل جميع العناصر باستخدام المرجع (بدون const)
    cout << "--- Changing all elements to 20 ---\n";
    for (int &i : num)
    {
        i = 20; // التعديل سينعكس مباشرة على الفيكتور الأصلي
        cout << i << " ";
    }
    cout << "\n\n";

    // 3. التعديل الفردي باستخدام الأقواس ودالة at
    num[1] = 40;     // تعديل العنصر الثاني
    num.at(2) = 80;  // تعديل العنصر الثالث
    num.at(4) = 90;  // تعديل العنصر الخامس

    // طباعة النتيجة النهائية
    cout << "--- Vector after individual changes ---\n";
    for (const int &i : num)
    {
        cout << i << " ";
    }
    cout << "\n";

    return 0;
}

```

</div>

💡 الخلاصة الهندسية:
تعود دائماً: إذا كانت حلقة التكرار for loop مخصصة للطباعة والقراءة فقط، استخدم const int &i. أما إذا كان هدفك هو تعديل القيم من الداخل، فاستخدم المرجع الحر int &i.

## 🔄 الدرس الثامن والأربعون: مكررات المتجه (Vector Iterators)

في هذا الدرس، سنتعرف على طريقة متقدمة للمرور على عناصر المتجه (Vector) باستخدام مفهوم يُسمى المكررات (Iterators).
المكرر (Iterator) هو ببساطة عبارة عن مؤشر (Pointer) مصمم خصيصاً للتنقل بين عناصر هياكل البيانات مثل المتجهات.

### 1. لماذا نستخدم الـ Iterators؟

رغم أننا نستطيع استخدام الـ for loop العادية للوصول للعناصر، إلا أن المكررات تُعتبر الأساس الذي تعتمد عليه الكثير من الدوال المتقدمة في C++، واستخدامها يجعلك تفهم وتطبق المفاهيم التي تعلمناها في فصل المؤشرات للوصول إلى العناوين والقيم بشكل احترافي.

### 2. طريقة تعريف المكرر (Declaration):

لكي نُعرف مكرراً، يجب أن يكون من نفس نوع البيانات للمتجه تماماً (إذا كان المتجه int يجب أن يكون المكرر int).

القاعدة (Syntax): vector<DataType>::iterator IteratorName;

### 3. دوال البداية والنهاية (begin و end):

لتحريك المؤشر على المتجه، نستخدم دالتين مهمتين:

.begin(): تُرجع مؤشراً يؤشر على عنوان أول عنصر في المتجه.

.end(): تُرجع مؤشراً يؤشر على المكان الذي يلي آخر عنصر (للتحقق من الوصول للنهاية).

💻 تطبيق عملي بالكود:

<div dir="ltr">

```cpp

#include <iostream>
#include <vector>
using namespace std;

int main()
{
    // 1. تعريف المتجه
    vector<int> num = {1, 2, 3, 4, 5};

    // 2. تعريف المكرر (Iterator) ليكون من نفس نوع المتجه
    vector<int>::iterator iter;

    cout << "--- Iterating using Vector Iterator ---\n";

    // 3. استخدام المكرر في حلقة for
    // يبدأ من أول عنصر، ويستمر طالما لم يصل للنهاية، ويزداد بمقدار 1 في كل لفة
    for (iter = num.begin(); iter != num.end(); iter++)
    {
        // 🚨 التريكة الهندسية: بما أن المكرر هو مؤشر (Pointer)
        // يجب استخدام المفتاح (*) لفك المرجعية (Dereferencing) وجلب القيمة لطباعتها
        cout << *iter << " ";
    }
    cout << endl;

    return 0;
}
```

</div>

💡 الخلاصة:
المكرر (Iterator) هو مؤشر ينتقل من بداية المتجه .begin() خطوة بخطوة iter++ حتى يصل للنهاية .end(). وللحصول على القيمة الحقيقية المخزنة داخل العنوان الذي يؤشر عليه في كل لفة، يجب أن نستخدم النجمة \*iter تماماً كما فعلنا في دروس المؤشرات!
