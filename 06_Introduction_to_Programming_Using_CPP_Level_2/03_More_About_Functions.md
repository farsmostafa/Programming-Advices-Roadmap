# الفصل الثالث: تعمق أكثر في الدوال (More About Functions)

## 📖 الدرس الرابع عشر: الإعلان عن الدوال مقابل تعريفها (Declaration vs Definition)

في لغة C++، الكومبايلر يقرأ الكود من أعلى إلى أسفل (Top-to-Bottom). بناءً على هذه القاعدة، ماذا يحدث لو قمنا باستدعاء دالة داخل main()، ولكننا كتبنا الكود الخاص بهذه الدالة تحت الـ main()؟

المشكلة:
سيقوم الكومبايلر بإصدار خطأ (Error: identifier not found)، لأنه عندما وصل إلى سطر الاستدعاء داخل الـ main، لم يكن يعرف بوجود هذه الدالة أصلاً!

الحل التقليدي:
أن نقوم بنسخ الدالة بالكامل ونضعها "فوق" دالة الـ main. (لكن هذا الحل غير عملي إذا كان لدينا 20 دالة، أو إذا كانت الدوال تستدعي بعضها البعض).

الحل الاحترافي (Declaration):
هنا يظهر الفرق بين مصطلحين في غاية الأهمية في هندسة البرمجيات:

### 1. تعريف الدالة (Function Definition / Implementation):

هو الكود الفعلي للدالة (البنية أو الـ Body) الذي يحتوي على المنطق والعمليات الحسابية.

### 2. الإعلان عن الدالة (Function Declaration / Prototype):

هو مجرد "عنوان" نضعه أعلى الكود (فوق الـ main) لُيخبر الكومبايلر: "يا كومبايلر، هناك دالة بهذا الاسم وهذه المدخلات ستجدها لاحقاً في الأسفل، فلا تُصدر خطأً عندما تراها".

طريقة كتابته: نأخذ السطر الأول فقط من الدالة (الهيدر)، ونضع في نهايته فاصلة منقوطة ;.

لا نحتاج حتى لكتابة أسماء المتغيرات، يكفي فقط تحديد نوعها.

💻 مثال عملي بالكود:

<div dir="ltr">

```cpp
#include <iostream>
using namespace std;

// 1. Function Declaration (الإعلان)
// نخبر الكومبايلر بوجود دالة اسمها Add تأخذ رقمين
void Add(int, int);

int main()
{
    // الآن يمكننا استدعاء الدالة بدون أي مشاكل
    Add(10, 20);
    return 0;
}

// 2. Function Definition (التعريف / جسم الدالة)
// الكود الفعلي مكتوب بالأسفل
void Add(int a, int b)
{
    cout << "Sum = " << a + b << endl;
}
```

</div>

💡 متى نستخدم الـ Declaration؟
نستخدمه بكثرة في المشاريع الكبيرة لتنظيم الكود، وعندما يكون لدينا دوال كثيرة تستدعي بعضها البعض ولا نريد أن نشغل بالنا بترتيب وضعها في الملف، فنجعل الإعلانات (Declarations) كلها في الأعلى، والتعريفات (Definitions) كلها في الأسفل.

## 🎛️ الدرس الخامس عشر: المعاملات الافتراضية (Default Parameters)

كلمة "Default" في البرمجة تعني "الخيار البديل في حالة الفشل". المعاملات الافتراضية (Default Parameters) هي أداة سحرية تجعل بعض مدخلات الدالة "اختيارية" (Optional).

فبدلاً من إجبار المستخدم أو المبرمج على إرسال قيم لكل المتغيرات التي تستقبلها الدالة، يمكنك إعطاء بعض هذه المتغيرات قيمة ابتدائية (مثلاً: 0)، ليقوم الكومبايلر باستخدامها تلقائياً إذا نسي المبرمج إرسالها!

💻 مثال عملي بالكود:

لنفترض أن لدينا دالة تجمع 4 أرقام. سنجعل أول رقمين إجباريين، وآخر رقمين اختياريين:

<div dir="ltr">

```cpp

#include <iostream>
using namespace std;

// المعاملات a و b إجبارية
// المعاملات c و d اختيارية (تم إعطاؤها قيمة افتراضية = صفر)
int MySum(int a, int b, int c = 0, int d = 0)
{
    return a + b + c + d;
}

int main()
{
    // الحالة الأولى: إرسال رقمين فقط (c و d سيعتبرهم الكومبايلر أصفار)
    cout << MySum(10, 20) << endl;         // النتيجة: 30

    // الحالة الثانية: إرسال 3 أرقام (d سيعتبره الكومبايلر صفراً)
    cout << MySum(10, 20, 30) << endl;     // النتيجة: 60

    // الحالة الثالثة: إرسال كل الأرقام (تتجاهل الدالة قيم الأصفار الافتراضية)
    cout << MySum(10, 20, 30, 40) << endl; // النتيجة: 100

    return 0;
}

```

</div>

🚨 ماذا لو لم نستخدم الديفولت بارامتر؟
لو قمنا بحذف الـ = 0 من المتغيرات c و d، ثم حاولنا استدعاء الدالة برقمين فقط MySum(10, 20)، سيغضب الكومبايلر ويصدر خطأً شهيراً:
(Too few arguments in function call)
يعني أنك أرسلت بيانات أقل من المطلوب لتشغيل الدالة.

💡 الفائدة الهندسية:
هذه الميزة تُستخدم بكثرة في بناء المكتبات (Libraries) والـ (APIs)، حيث تتيح للمبرمج استدعاء الدالة بأقل عدد ممكن من المدخلات للحالات البسيطة، مع إمكانية إضافة مدخلات إضافية للحالات المتقدمة دون الحاجة لإنشاء عدة دوال بأسماء مختلفة.

## 🎭 الدرس السادس عشر: التحميل الزائد للدوال (Function Overloading)

المشكلة:
تخيل أنك تريد برمجة دوال لجمع الأرقام؛ دالة تجمع رقمين من نوع (Integer)، ودالة تجمع رقمين من نوع (Double)، ودالة تجمع 3 أرقام، وهكذا. بالطريقة التقليدية ستضطر لتسمية كل دالة باسم مختلف (مثل: MySumTwoInts, MySumTwoDoubles, MySumThreeInts) مما يجعلك كمبرمج تتوه في حفظ الأسماء!

الحل السحري (Overloading):
في لغة C++، يمكنك تسمية كل هذه الدوال بنفس الاسم (مثلاً: MySum).

كيف يفرق الكومبايلر بينها؟
عن طريق ما يسمى بـ (التوقيع - Signature). الكومبايلر ذكي جداً، فهو ينظر إلى عدد ونوع المدخلات (Parameters) التي أرسلتها عند الاستدعاء، وبناءً عليها يقرر أي دالة سيقوم بتشغيلها.

💻 مثال عملي بالكود:

<div dir="ltr">

#include <iostream>

```cpp
using namespace std;

// الدالة الأولى: تجمع رقمين (Int)
int MySum(int a, int b)
{
    return a + b;
}

// الدالة الثانية: تجمع رقمين (Double)
double MySum(double a, double b)
{
    return a + b;
}

// الدالة الثالثة: تجمع 3 أرقام (Int)
int MySum(int a, int b, int c)
{
    return a + b + c;
}

int main()
{
    cout << MySum(10, 20) << endl;         // سيستدعي الدالة الأولى
    cout << MySum(10.5, 20.1) << endl;     // سيستدعي الدالة الثانية
    cout << MySum(10, 20, 30) << endl;     // سيستدعي الدالة الثالثة
    return 0;
}

```

</div>

🚨 قاعدة هامة (متى يغضب الكومبايلر؟):
الكومبايلر يفرق باختلاف (النوع أو العدد). لكن إذا قمت بتغيير أسماء المتغيرات فقط (مثلاً من a, b إلى x, y)، أو قمت بتغيير نوع الإرجاع (Return Type) فقط مع إبقاء المدخلات كما هي، سيصدر الكومبايلر خطأ يخبرك أن الدالة مكررة (Already has a body).

💡 معلومة متقدمة للمحترفين:
هذه الخاصية تعتبر نوعاً من أنواع (تعدد الأشكال - Polymorphism)، وهو مفهوم أساسي سندرسه بعمق في الـ Object Oriented Programming (OOP). يعني أن الكيان الواحد (اسم الدالة MySum) يمتلك عدة أشكال وسلوكيات تختلف حسب طريقة استخدامك له. (ميزة موجودة في C++ وليست متوفرة في كل لغات البرمجة).

## 📚 الدرس السابع عشر: مكدس الاستدعاءات (Call Stack / Call Hierarchy)

عندما تقوم ببناء برنامج كبير، نادراً ما تعمل دالة بمفردها. في الغالب، دالة main تستدعي Function 1، والذي بدوره يستدعي Function 2، وهكذا. هذا التسلسل الهرمي للاستدعاءات يُسمى (Call Stack) أو (Call Hierarchy).

كيف يعمل الـ Call Stack في الذاكرة؟

تخيل الـ Stack (المكدس) كأنه "رصّة أطباق". آخر طبق تضعه بالأعلى هو أول طبق تأخذه. هذه القاعدة تُسمى (LIFO: Last In, First Out).

رحلة تنفيذ البرنامج (خطوة بخطوة):

الدخول (Push): يبدأ البرنامج بدالة main، فيتم حجز مكان لها في الـ Stack وتصبح هي (الإطار النشط - Active Frame).

استدعت main دالة func1؟ تدخل func1 فوقها في الـ Stack وتصبح هي الإطار النشط، وتظل main في وضع "الانتظار".

استدعت func1 دالة func2؟ تدخل func2 في الأعلى... وهكذا.

الخروج (Pop): عندما تنتهي آخر دالة (مثلاً func4) من عملها، يتم "طردها" من الـ Stack (عملية Pop)، وتعود السيطرة (الإطار النشط) للدالة التي استدعتها (func3) لتكمل باقي الكود الخاص بها.

تستمر عملية الطرد (Pop) تراجعياً حتى نصل لـ main، وبانتهائها يفرغ الـ Stack وينتهي البرنامج.

💻 مثال برمجي للتوضيح:

<div dir="ltr">

```cpp
#include <iostream>
using namespace std;

void Func3() {
    cout << "I am Func 3" << endl; // ستنتهي وتُطرد أولاً
}

void Func2() {
    Func3(); // تستدعي Func3 وتنتظر
}

void Func1() {
    Func2(); // تستدعي Func2 وتنتظر
}

int main() {
    Func1(); // تستدعي Func1 وتنتظر
    return 0;
}

```

</div>

🔍 كيف تتبع الاستدعاءات في بيئة التطوير (IDE)؟
إذا كنت تقوم بعمل (Debugging) ووقفت عند نقطة توقف داخل Func3، وأردت أن تعرف "مَن الذي أوصلني إلى هنا؟":

• اضغط بزر الماوس الأيمن في الكود واختر (View Call Hierarchy) أو ابحث عن نافذة (Call Stack) (موجودة أسفل الشاشة في الـ VS Code).

• ستظهر لك قائمة تخبرك بالتسلسل العكسي: main نادت Func1 -> نادت Func2 -> نادت Func3.

• الفائدة الهندسية: هذه الأداة منقذة لحياتك كمبرمج عندما تعمل على مشروع ضخم يحتوي على آلاف الأسطر وتريد تتبع مسار الخطأ (Bug) أو فهم تسلسل عمل الكود المتداخل.

## 🛠️ الدرس الثامن عشر: أسرار وحيل للتعامل مع الدوال في بيئة التطوير (IDE Tips)

عندما يصبح مشروعك البرمجي ضخماً (آلاف الأسطر وعشرات الدوال)، فإن البحث اليدوي يصبح مضيعة للوقت. بيئات التطوير مثل (Visual Studio) و (VS Code) تقدم أدوات سحرية لزيادة إنتاجيتك (Productivity) وسرعتك في التعامل مع الدوال.

إليك أهم الأدوات والاختصارات التي يجب أن تحفظها عن ظهر قلب:

1. طي وتوسيع الكود (Collapse & Expand):
   بجوار كل دالة ستجد سهماً صغيراً (أو علامة -). اضغط عليه لطي الكود (إخفائه) لتنظيف الشاشة من الدوال التي انتهيت من برمجتها وتثق أنها تعمل بشكل سليم.

2. الذهاب إلى التعريف (Go To Definition - F12):
   إذا وجدت استدعاء لدالة (مثلاً MySum()) وأردت أن ترى الكود الخاص بها لتعديله، لا تبحث بعينك! اضغط عليها كليك يمين واختر Go To Definition (أو اضغط F12). سينقلك المحرر فوراً إلى مكان كتابة الدالة الأصلي.

3. الذهاب إلى الإعلان (Go To Declaration - Ctrl + F12):
   تنقلك إلى الهيدر أو الـ Prototype الخاص بالدالة الذي قمنا بشرحه في الدروس السابقة.

4. استراق النظر (Peek Definition - Alt + F12):
   بدلاً من أن يرسلك المحرر إلى مكان آخر بعيد عن الكود الذي تقرأه حالياً وتفقد تركيزك، اضغط Peek Definition. سيفتح لك نافذة صغيرة ومؤقتة تريك كود الدالة وتسمح لك بتعديله وأنت ما زلت تقف في مكانك!

5. البحث عن كل الاستدعاءات (Find All References - Shift + F12):
   إذا قررت إلغاء دالة معينة، يجب أن تعرف أولاً أين تم استدعاؤها حتى لا تكسر البرنامج. اضغط على اسم الدالة كليك يمين واختر Find All References، سيجلب لك قائمة بكل سطر في المشروع تم استخدام هذه الدالة فيه لتقوم بمراجعته.

6. إعادة التسمية الذكية (Rename - Ctrl + R, Ctrl + R):
   لو أردت تغيير اسم دالة من Func4 إلى koko، إياك أن تستخدم (Find and Replace) العادي! قد يغير كلمات مشابهة بالخطأ.
   استخدم Rename بالضغط كليك يمين على اسم الدالة، واكتب الاسم الجديد. سيتكفل المحرر الذكي بالبحث عن هذا الاسم بالتحديد وتغييره في كل الملفات الخاصة بالمشروع بأمان تام.

(ملاحظة: هذه الحيل ليست حكراً على C++، بل تنطبق على C# و Python و Java وأي لغة أخرى تستخدمها داخل بيئة الـ IDE، وهي علامة فارقة تميز المبرمج السريع من المبرمج المبتدئ).

## 🔁 الدرس التاسع عشر: الاستدعاء الذاتي (Recursion)

الـ (Recursion) هو من المواضيع "السهل الممتنع" في البرمجة. بكل بساطة: هو أن تقوم الدالة باستدعاء نفسها من داخل كودها الخاص!

كيف يعمل؟ (علاقة الـ Recursion بالـ Call Stack)

تخيل أنك تطلب من دالة أن تطبع الأرقام من 1 إلى 4.
في الـ Recursion، الدالة تطبع (1)، ثم تستدعي نفسها لتطبع (2)، ثم تستدعي نفسها لتطبع (3)، وهكذا.
كل استدعاء جديد يتم تخزينه كـ (إطار نشط - Active Frame) في الـ (Call Stack) الذي درسناه في الدرس 17. وعندما تصل لشرط التوقف (Base Case)، تبدأ الدوال بالخروج من الـ Stack تباعاً (عملية Pop).

⚠️ الخطر الأكبر: طفح الذاكرة (Stack Overflow)

الكثير من المبرمجين المبتدئين يستخدمون الـ Recursion بشكل عشوائي، ولكنهم يجهلون خطورته.
الـ (Call Stack) له مساحة محدودة جداً في الذاكرة (Memory). إذا قمت بعمل استدعاء ذاتي لعدد ضخم جداً من المرات، أو نسيت وضع "شرط توقف" صحيح للدالة، سيمتلئ الـ Stack تماماً ويفيض، مما يؤدي إلى انهيار البرنامج فوراً وظهور الخطأ الشهير: Stack Overflow.

💡 القاعدة الذهبية من المهندس:
99% من العمليات يمكن حلها باستخدام حلقات التكرار العادية (for أو while) وهي أكثر أماناً للذاكرة. لا تستخدم الـ Recursion إلا في الحالات المتقدمة التي يسهل فيها تبسيط اللوجيك المعقد وكنت واثقاً أنك لن تتخطى حجم الـ Stack.

💻 أمثلة عملية:

1. طباعة الأرقام تصاعدياً من N إلى M:

<div dir="ltr">

```cpp
#include <iostream>
using namespace std;

void PrintNumbers(int n, int m)
{
    // شرط التوقف (Base Case): طالما أن الرقم الأول أصغر أو يساوي الثاني
    if (n <= m)
    {
        cout << n << endl; // اطبع الرقم
        PrintNumbers(n + 1, m); // الدالة تستدعي نفسها وتزيد الرقم بمقدار 1
    }
}

int main()
{
    PrintNumbers(1, 4); // ستطبع: 1 2 3 4
    return 0;
}

```

</div>

2. حساب الأس (Power) باستخدام الـ Recursion:
   كيف تحسب $2^4$؟ هي عبارة عن $2 \times 2^3$.

<div dir="ltr">

```cpp
#include <iostream>
using namespace std;

int MyPower(int base, int power)
{
    // شرط التوقف: أي رقم أس صفر يساوي 1
    if (power == 0)
        return 1;
    else
        // اضرب الأساس في استدعاء الدالة لنفسها مع تقليل الأس بـ 1
        return base * MyPower(base, power - 1);
}

int main()
{
    cout << MyPower(2, 4) <<div endl; // النتيجة: 16
    return 0;
}

```

</div>
