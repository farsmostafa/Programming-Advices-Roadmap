# الفصل الرابع: المزيد عن المتغيرات (More About Variables)

## 🔒 الدرس العشرون: المتغيرات الساكنة (Static Variables)

لفهم المتغيرات الساكنة، يجب أولاً أن نفهم كيف تتصرف المتغيرات العادية داخل الدوال (Local Variables).

1. المتغير المحلي العادي (Normal Local Variable):

عندما تقوم بتعريف متغير داخل دالة (مثال: int number = 1;)، فإن الكومبايلر يقوم بإنشائه في الذاكرة عند استدعاء الدالة. ولكن بمجرد أن تنتهي الدالة من عملها، يتم تدمير هذا المتغير تماماً ومسحه من الذاكرة.
إذا قمت باستدعاء الدالة مرة أخرى، سيتم إنشاء المتغير من الصفر بقيمته الابتدائية (1).

نطاقه (Scope): داخل الدالة فقط.

دورة حياته (Lifetime): تنتهي بانتهاء الدالة.

2. المتغير الساكن (Static Variable):

بإضافة كلمة static قبل تعريف المتغير (مثال: static int number = 1;)، أنت تعطي أمراً صارماً للكومبايلر: "احجز مكاناً لهذا المتغير في الذاكرة مرة واحدة فقط، ولا تقم بتدميره عندما تنتهي الدالة، بل احتفظ بقيمته الأخيرة للاستدعاء القادم!".

نطاقه (Scope): داخل الدالة فقط (لا يمكن الوصول إليه من خارجها).

دورة حياته (Lifetime): طوال فترة عمل البرنامج (مثل المتغيرات العامة Global Variables، ولكن بأمان أكبر لأنه مخفي داخل الدالة).

💻 مقارنة عملية بالكود:

<div dir="ltr">

```cpp
#include <iostream>
using namespace std;

void MyFunction() 
{
    // متغير محلي عادي (يتم إنشاؤه وتدميره في كل استدعاء)
    int Number1 = 1; 

    // متغير ساكن (يتم إنشاؤه مرة واحدة فقط ويحتفظ بقيمته)
    static int Number2 = 1; 

    cout << "Normal: " << Number1 << " | Static: " << Number2 << endl;

    Number1++; // ستضيع هذه الزيادة بمجرد انتهاء الدالة
    Number2++; // سيتم الاحتفاظ بهذه الزيادة في الذاكرة
}

int main() 
{
    MyFunction(); // ستطبع: Normal: 1 | Static: 1
    MyFunction(); // ستطبع: Normal: 1 | Static: 2
    MyFunction(); // ستطبع: Normal: 1 | Static: 3

    return 0;
}
```
</div>

💡 الفائدة الهندسية:
نستخدم الـ static عندما نحتاج إلى متغير يقوم بحفظ "حالة" أو "تاريخ" معين عبر استدعاءات متكررة لنفس الدالة، دون الحاجة لتعريفه كـ (Global Variable) وتلويث النطاق العام للبرنامج. (مثال: دالة تقوم بحساب عدد المرات التي تم استدعاؤها فيها).


## 🤖 الدرس الحادي والعشرون: المتغيرات التلقائية (Automatic Variables / auto)

كما تعلمنا سابقاً، لغة C++ هي لغة قوية النوع (Strongly Typed)، بمعنى أنه يجب عليك كمبرمج أن تحدد نوع المتغير بدقة قبل استخدامه (int, double, bool) لتحديد حجم الذاكرة المستهلكة بدقة.

ولكن، لغة C++ توفر ميزة تسمى المتغيرات التلقائية (Automatic Variables) باستخدام الكلمة المحجوزة auto.

كيف تعمل؟

عند استخدام auto، أنت تطلب من الكومبايلر أن "يستنتج" نوع المتغير تلقائياً بناءً على القيمة الابتدائية التي تم إسنادها إليه (Initialization).

💻 مثال عملي بالكود:

<div dir="ltr">

```cpp
#include <iostream>
using namespace std;

int main() 
{
    auto x = 10;            // الكومبايلر سيفهم فوراً أنها (int)
    auto y = 12.5;          // الكومبايلر سيفهم أنها (double)
    auto z = "Abo Hadhoud"; // الكومبايلر سيفهم أنها (string)

    return 0;
}
```
</div>

🚨 تحذير شديد اللهجة (احذر استخدامها!):

المهندس أبو هدهود يُحذر بشدة من استخدام auto كبديل عن كتابة الأنواع الصريحة. لماذا؟

السيطرة على الذاكرة: المبرمج المحترف (C++ Developer) يجب أن يكون مسيطراً 100% على الذاكرة. إذا كان الرقم صغيراً يجب أن تعرفه كـ short لتوفر الذاكرة، وليس أن تترك الكومبايلر يقرر نيابة عنك فيحجزه كـ int.

البرمجة الكسولة: الاعتماد المفرط على auto يجعلك "مبرمجاً كسولاً" (أشبه بمبرمجي اللغات الديناميكية مثل Python)، ويُفقد لغة C++ أهم مميزاتها وهي الإدارة الدقيقة والمباشرة لموارد الجهاز.

الخلاصة: تمت دراسة هذا الموضوع فقط لكي تكون على علم به إذا واجهته صدفة أثناء قراءة أكواد مبرمجين آخرين، لكن إياك أن تعتمد عليه في كتابة برامجك الخاصة!

## 🗄️ الدرس الثاني والعشرون: متغيرات المسجل (Register Variables)

لفهم هذا النوع من المتغيرات، دعنا نسترجع سريعاً "هرم الذاكرة" (Memory Hierarchy) في الكمبيوتر من الأبطأ للأسرع:

القرص الصلب (Hard Disk - الأبطأ).

ذاكرة الوصول العشوائي (RAM - سريعة، وهي المكان الافتراضي لتخزين المتغيرات العادية).

ذاكرة التخزين المؤقت (Cache Memory - أسرع جداً).

مسجلات المعالج (CPU Registers - الأسرع على الإطلاق لأنها تقع داخل المعالج نفسه).

كيف كانت تعمل قديماً؟

في الإصدارات القديمة من C++، كان يمكنك تعريف متغير وتسبقه بكلمة register (مثال: register int x = 10;).
بهذا السطر، أنت تطلب من الكومبايلر ألا يقوم بتخزين هذا المتغير في الـ (RAM) كالمعتاد، بل يضعه مباشرة داخل مسجلات المعالج (CPU Registers) لضمان سرعة وصول قصوى. كانت تستخدم غالباً في العدادات (Counters) داخل الـ Loops التي تتطلب أداءً فائقاً.

🛑 لماذا تم إيقافها (Deprecated in C++11)؟

بدءاً من تحديث C++11 وما بعده، تم الاستغناء عن استخدام register وإلغاء تأثيرها العملي، لسببين هندسيين:

الندرة والتكلفة: مسجلات الـ CPU صغيرة جداً وغالية ومحدودة العدد، والتعامل المباشر معها "ليس مزحة" وقد يؤدي لتدهور أداء النظام إذا أساء المبرمج تقدير استخدامها.

ذكاء الكومبايلر الحديث: أصبحت مترجمات C++ الحديثة (Modern Compilers) ذكية جداً في عملية الـ (Optimization). الكومبايلر اليوم يعرف متى يضع متغيراً في الـ Register ومتى يضعه في الـ RAM بشكل تلقائي أفضل من المبرمج نفسه!

💡 الخلاصة:
لن تحتاج لاستخدام register في برامجك الحديثة، ولكن الهدف من هذا الدرس هو توسيع مداركك الهندسية لكي لا تتفاجأ إذا رأيت هذه الكلمة أثناء قراءتك لكود قديم (Legacy Code) في إحدى الشركات.
